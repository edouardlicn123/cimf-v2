## 服务层规范

### 服务设计原则

1. **静态方法为主**：使用 `@staticmethod`
2. **类型注解**：返回 `Optional[]`、`List[]`
3. **防御性编程**：`.first()` 调用后必须检查 None

```python
@staticmethod
def get_by_id(user_id: int) -> Optional[User]:
    user = User.objects.filter(id=user_id).first()
    return user  # 调用方需检查 None
```

## 表单层规范

### 表单设计原则

1. **HTML 直接渲染**：不依赖 Django Form 自动渲染
2. **统一 Widget**：`class="form-control"`
3. **CSRF Token**：所有 POST 表单必须包含 `{{ csrf_token }}`

```html
<form method="POST">
    {{ csrf_token }}
    <input type="text" name="field" class="form-control" required>
</form>
```

### 表单验证

```python
class UserEditForm(forms.Form):
    original_username = forms.CharField(widget=forms.HiddenInput())
    
    def clean_username(self):
        new_username = self.cleaned_data['username']
        original = self.cleaned_data['original_username']
        if new_username != original:
            raise ValidationError('无权修改用户名')
        return new_username
```

## Bug 易错点

### Python 代码

| 易错点 | 错误写法 | 正确写法 |
|--------|----------|----------|
| `.first()` 未检查 | `obj.method(); obj.name` | `obj = get(); if not obj: return` |
| `user_id` 为 None | `exclude(id=user_id)` | `if self.user_id: exclude()` |
| 整数字段搜索 | `filter(id__icontains=s)` | `try: filter(id=int(s))` |
| OR 查询 | `filter().filter()` | `filter(Q() \| Q())` |

### 模板语法

| 易错点 | 错误写法 | 正确写法 |
|--------|----------|----------|
| date + default | `{{ x\|date()\|default('-') }}` | `{{ x\|date() if x else '-' }}` |
| 外键访问 | `{{ obj.foreign.name }}` | `{{ obj.foreign.name if obj.foreign else '-' }}` |
| CSRF 调用 | `{{ csrf_token() }}` | `{{ csrf_token }}` |

### 三层架构一致性

**典型问题：choices 在多处定义**

```
models.py → forms.py → templates → JavaScript
```

**预防措施：**
1. 在 `core/constants.py` 中统一管理 choices
2. 表单和模板引用常量，不重复定义

```python
class UserRole:
    MANAGER = 'manager'
    CHOICES = [(MANAGER, '一类用户'), ...]

class UserForm(forms.Form):
    role = forms.ChoiceField(choices=UserRole.CHOICES)
```

### 复制代码注意事项

| 问题模式 | 预防措施 |
|----------|----------|
| 复制后忘改细节 | 仔细检查每个复制的字段 |
| 重构遗漏关联代码 | 使用 IDE 查找引用 |
| 前端防护后端未验证 | 后端必须独立验证 |

## 存储目录说明

项目使用 `storage/` 目录存放运行时数据，**无需手动创建**：

| 目录 | 用途 |
|------|------|
| `storage/logs/` | 日志文件（cimf.log, error.log, security.log） |
| `storage/uploads/` | 用户上传文件 |
| `storage/backups/` | 数据库备份 |

> 注意：`settings.py` 会在 Django 启动时自动创建这些目录。
