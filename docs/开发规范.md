## 服务层规范

### 服务设计原则

1. **静态方法为主**：使用 `@staticmethod`
2. **类型注解**：返回 `Optional[]`、`List[]`
3. **防御性编程**：`.first()` 调用后必须检查 None

```python
@staticmethod
def get_by_id(user_id: int) -> Optional[User]:
    user = User.objects.filter(id=user_id).first()
    return user  # 调用方需检查 None
```

## 视图层规范

### 认证规范（2026-04更新）

**全局中间件已启用**，但建议保留装饰器作为双重保护：

```python
# 页面视图（中间件会自动保护，但保留装饰器更清晰）
@login_required
def new_page_view(request):
    ...

# API 视图（必须保留，因为需要返回 JSON 错误而非重定向）
@login_required_json
def api_new_endpoint(request):
    ...

@admin_required
def api_admin_action(request):
    ...
```

**认证要求：**
1. **全局中间件已启用**：除白名单外，所有页面需登录
2. **API 必须使用装饰器**：
   - 页面视图：`@login_required`（可选，中间件已保护）
   - JSON API：`@login_required_json`（必须，返回 JSON 错误）
   - 管理员 API：`@admin_required`（必须，返回 JSON 错误）
3. **运行检查命令**：新增功能后必须运行
   ```bash
   ./venv/bin/python manage.py check
   ```
   确保没有 `CIMF_W001` 警告
4. **CSRF 保护**：所有 POST 表单使用 `{{ csrf_token }}`

### 视图结构

```python
@login_required
def page_view(request):
    if not PermissionService.can_access_admin(request.user):
        messages.error(request, '需要管理员权限')
        return redirect('core:dashboard')
    
    return render(request, 'path/to/template.html', context)
```

## 表单层规范

### 表单设计原则

1. **Django Form + BootstrapFormMixin**：所有表单继承 `forms.Form` 并混入 `BootstrapFormMixin`
2. **统一样式**：`BootstrapFormMixin` 自动为所有字段添加 `class="form-control"`
3. **CSRF Token**：所有 POST 表单必须包含 `{{ csrf_token }}`
4. **模板渲染**：使用 `{{ form.as_div() }}` 或手动渲染字段

```html
<form method="POST" class="form-horizontal">
    {{ csrf_token }}
    {{ form.as_div() }}
    <button type="submit" class="btn btn-primary">提交</button>
</form>
```

### 表单基类

```python
from core.mixins import BootstrapFormMixin

class UserEditForm(forms.Form, BootstrapFormMixin):
    username = forms.CharField(label='用户名', max_length=150)
    email = forms.EmailField(label='邮箱', required=False)
```

### 表单验证

```python
class UserEditForm(forms.Form):
    original_username = forms.CharField(widget=forms.HiddenInput())
    
    def clean_username(self):
        new_username = self.cleaned_data['username']
        original = self.cleaned_data['original_username']
        if new_username != original:
            raise ValidationError('无权修改用户名')
        return new_username
```

## Bug 易错点

### Python 代码

| 易错点 | 错误写法 | 正确写法 |
|--------|----------|----------|
| `.first()` 未检查 | `obj.method(); obj.name` | `obj = get(); if not obj: return` |
| `user_id` 为 None | `exclude(id=user_id)` | `if self.user_id: exclude()` |
| 整数字段搜索 | `filter(id__icontains=s)` | `try: filter(id=int(s))` |
| OR 查询 | `filter().filter()` | `filter(Q() \| Q())` |

### 模板语法

| 易错点 | 错误写法 | 正确写法 |
|--------|----------|----------|
| date + default | `{{ x\|date()\|default('-') }}` | `{{ x\|date() if x else '-' }}` |
| 外键访问 | `{{ obj.foreign.name }}` | `{{ obj.foreign.name if obj.foreign else '-' }}` |
| CSRF 调用 | `{{ csrf_token() }}` | `{{ csrf_token }}` |

### 三层架构一致性

**典型问题：choices 在多处定义**

```
models.py → forms.py → templates → JavaScript
```

**预防措施：**
1. 在 `core/constants.py` 中统一管理 choices
2. 表单和模板引用常量，不重复定义

```python
class UserRole:
    MANAGER = 'manager'
    CHOICES = [(MANAGER, '一类用户'), ...]

class UserForm(forms.Form):
    role = forms.ChoiceField(choices=UserRole.CHOICES)
```

### 复制代码注意事项

| 问题模式 | 预防措施 |
|----------|----------|
| 复制后忘改细节 | 仔细检查每个复制的字段 |
| 重构遗漏关联代码 | 使用 IDE 查找引用 |
| 前端防护后端未验证 | 后端必须独立验证 |

| 目录 | 用途 |
|------|------|
