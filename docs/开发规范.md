# 开发规范

## 表单提交 CSRF 防护

### 重要：所有表单提交必须包含 CSRF token

为了防止 CSRF 攻击，所有 POST、PUT、DELETE 请求必须包含 CSRF token。

### 常规表单（同步提交）

使用 Flask-WTF 的 `form.hidden_tag()` 自动生成隐藏字段：

```html
<form method="POST">
    {{ form.hidden_tag() }}
    <!-- 或 -->
    <input type="hidden" name="csrf_token" value="{{ csrf_token() }}"/>
    <!-- 表单内容 -->
</form>
```

### AJAX/Fetch 请求

使用 Fetch API 发送异步请求时，需要手动添加 CSRF token：

```javascript
// 方法1：添加到 FormData 中
const formData = new FormData();
formData.append('csrf_token', '{{ csrf_token() }}');
formData.append('field1', value1);

fetch('/api/endpoint', {
    method: 'POST',
    headers: {'Content-Type': 'application/x-www-form-urlencoded'},
    body: new URLSearchParams(formData)
})

// 方法2：添加到请求头
fetch('/api/endpoint', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/x-www-form-urlencoded',
        'X-CSRFToken': '{{ csrf_token() }}'
    },
    body: new URLSearchParams(formData)
})
```

### 常见遗漏位置

- 删除操作的表单
- 状态切换（如启用/禁用）
- 表格内的行操作
- AJAX 异步提交

### 检查清单

添加新表单或 AJAX 请求时，确保：
- [ ] 同步表单使用了 `form.hidden_tag()` 或 `csrf_token()`
- [ ] AJAX 请求在 FormData 中包含 `csrf_token`
- [ ] DELETE 请求同样需要 CSRF token（不能仅靠 method: 'DELETE'）
