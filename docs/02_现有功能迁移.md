# 现有功能迁移计划

## 一、概述

将现有 Flask 应用中的功能模块迁移到 `app/modules/` 目录下，实现模块化架构。

## 二、当前结构

```
app/
├── routes/
│   ├── auth.py      # 认证
│   ├── admin.py     # 系统管理
│   ├── main.py      # 主路由（含 dashboard/profile）
│   └── export.py    # 导出
├── services/
│   ├── auth_service.py
│   ├── user_service.py
│   ├── permission_service.py
│   └── settings_service.py
├── forms/
├── templates/
└── ...
```

## 五、路由注册

在 `app/routes/__init__.py` 中注册各模块 Blueprint：

```python
def register_blueprints(app):
    # Core 模块
    from app.modules.core.auth import auth_bp
    from app.modules.core.admin import admin_bp
    from app.modules.core.workspace import workspace_bp
    
    # 其他模块
    from app.modules.profile import profile_bp
    from app.modules.export import export_bp
    
    app.register_blueprint(auth_bp)
    app.register_blueprint(admin_bp)
    app.register_blueprint(workspace_bp)
    app.register_blueprint(profile_bp)
    app.register_blueprint(export_bp)
```

## 六、实施步骤

### Step 1: 创建目录结构

```bash
mkdir -p app/modules/core/auth/templates
mkdir -p app/modules/core/admin/templates
mkdir -p app/modules/core/workspace/templates
mkdir -p app/modules/profile/templates
mkdir -p app/modules/export/templates
```

### Step 2: 迁移 Auth 模块

1. 复制 `routes/auth.py` → `modules/core/auth/routes.py`
2. 调整导入路径
3. 创建 `modules/core/auth/__init__.py`
4. 移动模板文件

### Step 3: 迁移 Admin 模块

1. 复制 `routes/admin.py` → `modules/core/admin/routes.py`
2. 调整导入路径
3. 创建 `modules/core/admin/__init__.py`
4. 移动模板文件

### Step 4: 迁移 Workspace 模块

1. 从 `main.py` 提取 dashboard 相关代码
2. 创建 `modules/core/workspace/routes.py`
3. 创建 `modules/core/workspace/__init__.py`

### Step 5: 迁移 Profile 模块

1. 从 `main.py` 提取 profile/settings 相关代码
2. 创建 `modules/profile/routes.py`
3. 创建 `modules/profile/__init__.py`

### Step 6: 迁移 Export 模块

1. 复制 `routes/export.py` → `modules/export/routes.py`
2. 创建 `modules/export/__init__.py`

### Step 7: 更新路由注册

修改 `app/routes/__init__.py` 统一注册所有 Blueprint。

### Step 8: 测试验证

- 启动应用
- 验证所有路由正常工作
- 验证模板渲染正确

## 七注意事项

1. **保持现有功能不变** - 迁移过程中不修改业务逻辑
2. **调整导入路径** - 确保模块间导入正确
3. **移动模板文件** - 将相关模板移到模块目录下
4. **更新静态资源引用** - 确保 CSS/JS 路径正确
5. **保持 URL 兼容** - 尽量保持原有 URL 不变，或做好重定向
