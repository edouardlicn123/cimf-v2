### 2.1 模板命名规范（重要）

当多个应用的模板目录在 `DIRS` 中配置时，Django 会按顺序查找。可能出现同名的 `list.html`、`edit.html`、`view.html` 被错误匹配的问题。

**解决方案**：
1. **推荐**：为不同应用使用带后缀的唯一模板名，如 `list.html`、`edit.html`
2. **备选**：在视图层使用 `select_template()` 显式指定模板路径

```python
# 视图中的正确用法
from django.template.loader import select_template

return render(request, 'list.html', context)

# 或使用 select_template
tpl = select_template(['customer/list.html', 'list.html'])
return render(request, tpl.template.name, context)
```

## 4. Jinja2 语法规范

### 4.1 过滤器语法（必须使用括号）

| 场景 | ✅ 正确 | ❌ 错误 |
|------|---------|--------|
| 默认值 | `{{ x\|default("y") }}` | `{{ x\|default:"y" }}` |
| 日期格式化 | `{{ d\|date("Y-m-d") }}` | `{{ d\|date:"Y-m-d" }}` |
| JSON 转义 | `{{ x\|tojson }}` | - |

### 4.2 CSRF Token

**模板中使用：**

| ✅ 正确 | ❌ 错误 |
|---------|---------|
| `{{ csrf_token }}` | `{% csrf_token %}` |

**实现机制：**

`csrf_token` 在 Jinja2 环境中返回完整的隐藏 input 元素：

```python
# cimf_django/context_processors.py
def csrf_token(request):
    from django.middleware.csrf import get_token
    from django.utils.safestring import mark_safe
    token = get_token(request)
    return {'csrf_token': mark_safe(f'<input type="hidden" name="csrfmiddlewaretoken" value="{token}">')}
```

**常见问题排查：**

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 显示原始 token 字符串 | 返回的是纯字符串而非 HTML | 使用 `mark_safe()` 标记 |
| HTML 被转义显示 | 输出被 HTML 转义 | 确保返回 `mark_safe()` 对象 |
| 页面显示乱码 | `{{ csrf_token }}` 输出未处理的字符串 | 确认 context processor 返回值正确 |

### 4.3 字典遍历

```html
{% for k, v in dict.items() %}
    {{ k }}: {{ v }}
{% endfor %}
```

**注意**：必须加 `()` 调用方法

### 4.4 三元表达式

```html
{{ value if condition else default }}
```

### 4.5 过滤器链式调用方法

```html
<!-- ❌ 错误：方法不生效 -->
{{ x|default("y").split(",") }}

<!-- ✅ 正确：加括号分组 -->
{{ (x|default("y")).split(",") }}
```

### 4.6 常用过滤器速查

| 过滤器 | 用途 | 示例 |
|--------|------|------|
| `default` | 默认值 | `{{ value\|default("-") }}` |
| `default_if_none` | 仅 None 时默认值 | `{{ value\|default_if_none("无") }}` |
| `date` | 日期格式化 | `{{ value\|date("Y-m-d H:i") }}` |
| `truncatechars` | 按字符数截断 | `{{ text\|truncatechars(50) }}` |
| `slice` | 字符串切片 | `{{ text\|slice(0, 50) }}` |
| `escape` | HTML 转义 | `{{ html\|escape }}` |
| `tojson` | JSON 转义 | `{{ data\|tojson }}` |
| `safe` | 标记为安全 HTML | `{{ html\|safe }}`（慎用） |
| `length` | 长度 | `{{ list\|length }}` |
| `replace` | 替换 | `{{ text\|replace("a", "b") }}` |
| `filesizeformat` | 文件大小 | `{{ size\|filesizeformat }}` |

### 9.1 现有可复用组件

### 9.2 组件设计原则

3. **无依赖**：不依赖页面特定的全局变量
4. **可复用**：至少在 3 个地方使用才考虑抽取

### 9.3 适合提取为组件的场景

- 重复出现 3 次以上的 UI 模式
- 逻辑复杂且独立的功能
- 需要维护状态的交互组件

## 10. 性能优化

### 10.1 查询优化

**问题**：N+1 查询

```python
# ❌ N+1 查询
customers = Customer.objects.all()
# 模板中访问 customer.customer_type.name 会为每条数据触发额外查询

# ✅ 使用 select_related
customers = Customer.objects.select_related(
    'customer_type',
    'country',
    'enterprise_type',
    'customer_level'
).all()
```

**预计算统计：**

```html
<!-- ❌ 每次访问都触发 count() 查询 -->
{{ taxonomy.items.count }}

<!-- ✅ 在视图层预计算 -->
taxonomies = Taxonomy.objects.prefetch_related('items')
for t in taxonomies:
    t.item_count = t.items.count()
```

### 10.2 渲染优化

| 优化项 | 说明 |
|--------|------|
| 在视图层过滤数据 | 避免在模板中使用复杂的 `{% if %}` 过滤 |
| 使用 `only()` | 只选择需要的字段，减少内存占用 |
| 分页 | 大数据集必须分页，默认每页 20-50 条 |
| 避免模板中调用方法 | 模板中调用方法会触发额外查询 |

### 10.3 缓存策略

- 静态资源使用版本号：`{{ static('js/app.js') }}?v={{ timestamp }}`
- 不缓存用户相关数据
- 使用 Django 缓存框架缓存复杂计算结果

## 13. 反模式

### ❌ 禁止：直接输出用户输入到 JavaScript

```html
<!-- 危险：XSS 漏洞 -->
<script>
    var name = "{{ user.name }}";  // 如果 name 包含引号会破坏 JS
    var id = {{ user.id }};         // 如果 id 为空会报错
</script>

<!-- ✅ 安全：使用 tojson -->
<script>
    var user = {{ user|tojson }};
</script>
```

### ❌ 禁止：在模板中执行复杂计算

```html
<!-- 危险：逻辑复杂，难以维护 -->
{{ (a + b) * c if x > 0 else d }}

<!-- ✅ 安全：在视图层处理 -->
{{ calculated_value }}
```

### ❌ 禁止：嵌套过深的条件判断

```html
<!-- 危险：可读性差 -->
{% if a %}
    {% if b %}
        {% if c %}
            {{ value }}
        {% endif %}
    {% endif %}
{% endif %}

<!-- ✅ 安全：使用 {% with %} -->
{% with value %}
```

### ❌ 禁止：硬编码 URL

```html
<!-- 危险：维护困难 -->
<a href="/admin/users/">用户</a>

<!-- ✅ 安全：使用 url() -->
<a href="{{ url('core:users') }}">用户</a>
```

### ❌ 禁止：Modal 放在表格内部

```html
<!-- 危险：HTML 结构错误 -->
<table>
    ...
    <div class="modal">...</div>
</table>

<!-- ✅ 安全：放在表格外 -->
<table>
    ...
</table>
<div class="modal">...</div>
```

### ❌ 禁止：外键访问无空值检查

```html
<!-- 危险：customer.country 可能为空 -->
{{ customer.country.name }}

<!-- ✅ 安全：逐层检查 -->
{% if customer.country %}
    {{ customer.country.name }}
{% else %}
    -
{% endif %}
```

### ❌ 禁止：在条件判断中使用可能为空的对象

```html
<!-- 危险：customer 可能为空（新建页面） -->
{% if customer.enterprise_type_id == item.id %}

<!-- ✅ 安全 -->
{% if customer and customer.enterprise_type_id == item.id %}
```

### ❌ 禁止：使用字符串比较 Django 消息级别

```html
<!-- 危险：message.level 是整数，不是字符串 -->
{% if message.tags == "success" %}

<!-- ✅ 安全：使用整数或 message.tags -->
{% if message.level == 25 %}
```

### ❌ 禁止：在模板中使用 QuerySet 方法不带括号

```html
<!-- 危险：count 是方法不是属性 -->
{{ taxonomy.items.count }}

<!-- ✅ 正确：count 是方法，需要括号 -->
{{ taxonomy.items.count() }}

<!-- ✅ 最佳方案：在视图层预计算 -->
{% for taxonomy in taxonomies %}
    <span>{{ taxonomy.item_count }}</span>
{% endfor %}
```

| `default` 过滤器 | `{{ x\|default("y") }}`（括号） |
| `date` 过滤器 | `{{ d\|date("Y-m-d") }}`（括号） |
| `csrf_token` | `{{ csrf_token }}`（变量，非 tag） |
| `dict.items()` | 必须加括号 |

### 15.2 空值处理

| 检查项 | 处理方式 |
|--------|----------|
| 用户输入变量 | `{{ value\|default("-") }}` |
| 外键访问 | `{% if obj.relation %}{{ obj.relation.name }}{% else %}-{% endif %}` |
| 表单中对象属性 | `{% if obj and obj.attr == value %}` |
| 日期字段 | `{{ date\|date("Y-m-d H:i") }}` |

### 15.3 HTML 结构

| 检查项 | 规范 |
|--------|------|
| Modal 位置 | 必须放在 `<table>` 外部 |
| 标签闭合 | 确保所有标签正确闭合 |
| colspan | 每行 colspan 总和必须一致 |

### 15.4 表单安全

| 检查项 | 规范 |
|--------|------|
| CSRF | 所有 POST 表单包含 `{{ csrf_token }}` |
| XSS | JS 中使用 `{{ value\|tojson }}` |
| data 属性 | 使用 `{{ value\|escape }}` |

### 16.4 CSRF Token 实现

`csrf_token` 在 Jinja2 中返回完整的隐藏 input 元素：

```python
# cimf_django/context_processors.py
def csrf_token(request):
    from django.middleware.csrf import get_token
    from django.utils.safestring import mark_safe
    token = get_token(request)
    return {'csrf_token': mark_safe(f'<input type="hidden" name="csrfmiddlewaretoken" value="{token}">')}
```

**使用方式**：

```html
<form method="post">
    {{ csrf_token }}  <!-- 直接输出隐藏 input -->
</form>
```
