## 一、概述

### 1.1 初始化流程定义

系统初始化是指首次部署或重建数据库时，执行 `init_db.py` 脚本完成基础数据准备的过程。

**初始化与模块安装的区别：**
- **初始化（init_db.py）**：创建核心表结构、系统配置、管理员账号、注册模块（不安装）
- **模块安装（模块管理页面）**：用户手动触发，执行模块migrate、创建词汇表、生成示例数据

### 1.2 初始化四阶段

| 阶段 | 名称 | 主要内容 | 是否可跳过 |
|------|------|----------|------------|
| 阶段1 | 数据库结构 | 执行Django migrations | ❌ 不可 |
| 阶段2 | 系统配置 | 设置、权限、词汇表、邮件模板 | ❌ 不可 |
| 阶段3 | 用户管理 | 创建管理员账号 | ❌ 不可 |
| 阶段4 | 业务模块 | 扫描并注册模块（不安装） | ❌ 不可 |

### 1.3 性能目标

| 场景 | 目标时间 | 当前时间 |
|------|----------|----------|
| 完整初始化（--with-data） | < 15秒 | ~30秒 |
| 增量初始化（--incremental） | < 5秒 | ~10秒 |
| 仅结构初始化（无--with-data） | < 5秒 | ~8秒 |

## 二、阶段详解与规范

### 2.1 阶段1：数据库结构（migrations）

**目的：** 创建所有核心数据表

**执行命令：**
```python
from django.core.management import call_command
call_command('migrate', '--noinput')
```

**✅ 规范（必须遵守）：**

1. **禁止使用subprocess调用migrate**
   ```python
   # ❌ 错误示例（慢，每次启动新进程）
   subprocess.run([python, '-c', 'call_command("migrate")'])
   
   # ✅ 正确示例（快，直接调用）
   call_command('migrate', '--noinput')
   ```

2. **检查pending migrations后再执行**
   ```python
   def _has_pending_migrations() -> bool:
       from django.core.management import call_command
       from io import StringIO
       out = StringIO()
       call_command('showmigrations', '--plan', stdout=out)
       return any(line.startswith('[ ]') for line in out.getvalue().strip().split('\n'))
   ```

3. **模块migrate必须通过call_command调用**
   - 位置：`core/node/services/module_service.py`
   - 必须临时将模块加入INSTALLED_APPS后调用
   - 必须在finally块中恢复原始INSTALLED_APPS

**性能优化建议：**
- 合并migrations减少文件数量（`./manage.py squashmigrations`）
- 避免在migrations中执行耗时操作

### 2.2 阶段2：系统配置初始化

**目的：** 初始化系统运行所需的基础数据

**包含4个子任务：**

#### 2.2.1 系统设置（SettingsService）
```python
from core.services import SettingsService
SettingsService.reset_to_default()
```

**✅ 规范：**
- 必须使用 `bulk_create`（已有）
- 必须先查询existing_keys，避免重复插入
- 必须清除缓存 `clear_cache()`

#### 2.2.2 角色权限（PermissionService）
```python
from core.services import PermissionService
PermissionService.init_default_role_permissions()
```

**✅ 规范：**
- 必须批量查询 `existing_keys`（已有）
- 必须检查 `setting_key not in existing_keys` 后插入
- 权限定义必须在 `PERMISSIONS` 常量中维护

#### 2.2.3 词汇表（TaxonomyService）⚡主要性能瓶颈
```python
from core.services.taxonomy_service import TaxonomyService
TaxonomyService.init_default_taxonomies()
```

**✅ 规范（重要）：**
1. **必须一次性加载所有已存在词汇项**（避免37次查询）
   ```python
   # ❌ 错误（慢）
   for tax in DEFAULT_TAXONOMIES:
       existing = set(taxonomy.items.values_list('name', flat=True))
   
   # ✅ 正确（快）
   existing_items = defaultdict(set)
   for item in TaxonomyItem.objects.all().values('taxonomy__slug', 'name'):
       existing_items[item['taxonomy__slug']].add(item['name'])
   ```

2. **必须使用bulk_create批量插入**
   ```python
   if items_to_create:
       TaxonomyItem.objects.bulk_create(items_to_create, batch_size=500)
   ```

3. **必须在transaction.atomic()中执行**
   ```python
   from django.db import transaction
   with transaction.atomic():
       # 所有创建操作
   ```

4. **词汇表数量**：当前37个，新增必须更新此文档

#### 2.2.4 邮件模板（TemplateService）
```python
from core.smtp.services.template_service import TemplateService
TemplateService.init_default_templates()
```

**✅ 规范：**
- 必须使用 `bulk_create`
- 必须检查模板是否已存在

**性能关键点：**
- 词汇表初始化占总时间的40-50%，优化后预计节省5-8秒

### 2.3 阶段3：用户管理

**目的：** 创建系统管理员账号

**✅ 规范：**

1. **密码复杂度要求**
   - 开发环境：无限制（默认admin123）
   - 生产环境：≥10位，必须设置 `ALLOW_SEED_PROD=1`

2. **管理员账号配置（通过环境变量）**
   ```bash
   ADMIN_USERNAME=admin
   ADMIN_PASSWORD=admin123  # 生产环境必须≥10位
   ADMIN_EMAIL=admin@example.com
   ADMIN_THEME=default
   ```

3. **默认导航卡片**
   - 必须定义 `DEFAULT_NAV_CARDS` 常量
   - 包含6个默认卡片（必应、豆包、千问等）

4. **权限设置**
   - `is_admin=True`
   - `role=UserRole.MANAGER`

### 2.4 阶段4：业务模块

**目的：** 扫描模块目录并注册到数据库（不安装）

**✅ 规范：**

1. **仅注册，不安装**
   ```python
   # ✅ 正确：仅注册到Module表
   module = Module.objects.create(...)
   module.is_installed = False  # 关键！
   
   # ❌ 错误：初始化时安装模块
   ModuleService.install_module(module_id)
   ```

2. **模块扫描逻辑**
   - 扫描 `modules/` 目录
   - 读取每个模块的 `module.py`
   - 解析 `MODULE_INFO` 字典
   - 注册到 `Module` 模型

3. **跳过已注册模块**
   ```python
   pending_modules = [
       m for m in all_modules
       if not (m.get('is_registered') and m.get('is_installed'))
   ]
   ```

## 三、服务层init函数规范

### 3.1 TaxonomyService.init_default_taxonomies()

**文件：** `core/services/taxonomy_service.py`

**规范检查清单：**
- [ ] 是否一次性加载所有已存在词汇项？
- [ ] 是否使用 `bulk_create` 批量插入？
- [ ] 是否在 `transaction.atomic()` 中执行？
- [ ] 是否检查数据已存在避免重复？
- [ ] 是否处理37个词汇表（含100+国家）？

**优化后代码示例：**
```python
@staticmethod
def init_default_taxonomies() -> int:
    """初始化预置分类数据（优化版：批量查询+批量插入）"""
    from core.models import Taxonomy, TaxonomyItem
    from django.db import transaction
    from collections import defaultdict
    
    # 一次性加载所有已存在的词汇表
    existing_taxonomies = {t.slug: t for t in Taxonomy.objects.all()}
    
    # 一次性加载所有已存在的词汇项
    existing_items = defaultdict(set)
    for item in TaxonomyItem.objects.all().values('taxonomy__slug', 'name'):
        existing_items[item['taxonomy__slug']].add(item['name'])
    
    count = 0
    with transaction.atomic():
        for tax in DEFAULT_TAXONOMIES:
            slug = tax['slug']
            
            # 创建或获取词汇表
            if slug in existing_taxonomies:
                taxonomy = existing_taxonomies[slug]
            else:
                taxonomy = Taxonomy.objects.create(
                    slug=slug,
                    name=tax['name'],
                    description=tax.get('description', '')
                )
                existing_taxonomies[slug] = taxonomy
                count += 1
            
            # 批量检查并创建词汇项
            existing_names = existing_items.get(slug, set())
            items_to_create = []
            
            for idx, item_name in enumerate(tax['items']):
                if item_name not in existing_names:
                    items_to_create.append(TaxonomyItem(
                        taxonomy=taxonomy,
                        name=item_name,
                        weight=idx
                    ))
                    existing_names.add(item_name)  # 更新内存缓存
            
            if items_to_create:
                TaxonomyItem.objects.bulk_create(items_to_create, batch_size=500)
    
    return count
```

### 4.3 修改 module_service.py 检查项

- [ ] 是否消除subprocess调用？（必须）
- [ ] 是否改用 `call_command` 直接调用？（必须）

**优化后代码示例：**
```python
@staticmethod
def _run_migration_inline(module_id: str, app_name: str) -> list:
    """以内联方式执行迁移，避免subprocess"""
    from django.core.management import call_command
    from django.conf import settings
    from django.apps import apps
    import logging
    
    logger = logging.getLogger(__name__)
    errors = []
    
    # 保存原始状态
    original_apps = list(settings.INSTALLED_APPS)
    
    try:
        # 临时将模块加入INSTALLED_APPS
        if app_name not in settings.INSTALLED_APPS:
            settings.INSTALLED_APPS.append(app_name)
            # 重新初始化apps
            apps.app_configs = {}
            apps.ready = False
            apps.populate(settings.INSTALLED_APPS)
        
        # 执行迁移
        call_command('migrate', module_id, verbosity=1, interactive=False)
        logger.info(f"模块 {module_id} 迁移完成")
        
    except Exception as e:
        error_msg = str(e)
        logger.error(f"模块 {module_id} 迁移失败: {error_msg}")
        errors.append(f'migrate 失败: {error_msg}')
        
    finally:
        # 恢复原始INSTALLED_APPS
        settings.INSTALLED_APPS = original_apps
        apps.app_configs = {}
        apps.ready = False
        apps.populate(settings.INSTALLED_APPS)
    
    return errors
```

## 五、性能优化规范

### 5.1 禁止的做法（❌）

| 做法 | 原因 | 替代方案 |
|------|------|----------|
| 使用subprocess执行Django命令 | 每次启动新进程，慢10倍+ | 使用 `call_command()` |
| 逐条插入数据库记录（for+save） | 每次插入一次DB交互 | 使用 `bulk_create()` |
| 在循环中查询数据库（N+1） | 37个词汇表=37次查询 | 一次性加载到内存 |
| 重复执行已完成的迁移 | 浪费时间 | 检查 `_has_pending_migrations()` |
| 在初始化时安装模块 | 偏离设计原则 | 由用户手动安装 |

---

### 5.2 推荐的做法（✅）

| 做法 | 场景 | 效果 |
|------|------|------|
| `call_command()` | 执行Django命令 | 避免subprocess开销，提速80% |
| `bulk_create()` | 批量插入数据 | 减少DB交互，提速60% |
| `transaction.atomic()` | 包装批量操作 | 保证一致性，提速20% |
| 一次性加载数据 | 检查数据是否存在 | 减少查询次数，提速50% |
| `exists()` / `count()` | 判断数据是否存在 | 避免加载全部数据 |

**代码示例：**
```python
# ✅ 推荐：综合使用多种优化手段
from django.db import transaction
from collections import defaultdict

# 1. 一次性加载（减少查询）
existing_items = defaultdict(set)
for item in TaxonomyItem.objects.all().values('taxonomy__slug', 'name'):
    existing_items[item['taxonomy__slug']].add(item['name'])

# 2. 在事务中批量操作
with transaction.atomic():
    # 3. 批量创建
    if items_to_create:
        TaxonomyItem.objects.bulk_create(items_to_create, batch_size=500)
```

## 七、常见问题（FAQ）

### Q1：为什么初始化时不安装模块？
**A：** 初始化是系统级操作，只准备运行环境。模块安装是用户级操作，需要用户手动触发（模块管理页面），这样设计更灵活、更安全。

### Q2：如何加快词汇表初始化？
**A：** 
1. 一次性加载所有已存在词汇项（避免N次查询）
2. 使用 `bulk_create` 批量插入
3. 在 `transaction.atomic()` 中执行
4. 考虑将固定数据导出为fixture，用 `loaddata` 加载

### Q3：subprocess和call_command的区别？
**A：**
- `subprocess`：启动新Python进程，重新初始化Django，开销大（~3秒/次）
- `call_command`：直接在当前进程调用，无需重新初始化，开销小（~0.5秒/次）

### Q4：如何添加新的词汇表？
**A：** 
1. 在 `core/services/taxonomy_service.py` 的 `DEFAULT_TAXONOMIES` 中添加定义
2. 更新本文档2.2.3节的词汇表数量
3. 运行 `init_db.py --with-data` 测试

### Q5：生产环境初始化有什么不同？
**A：**
- 必须设置 `ALLOW_SEED_PROD=1` 才能执行
- 管理员密码必须≥10位
- 不生成示例数据
- 建议使用环境变量传递配置
