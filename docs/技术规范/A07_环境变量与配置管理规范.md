# 环境变量与配置管理规范

> 文档版本：1.0  
> 创建日期：2026-05-02  
> 适用范围：cimf_django/settings.py、config.env、config.env.sample

---

## 一、概述

### 1.1 目的

本规范定义 CIMF 项目中环境变量和配置文件的管理标准，实现：
- 开发/生产环境配置分离
- 敏感信息不进入版本控制
- 配置项有清晰的文档说明
- 便于部署和运维

### 1.2 配置文件结构

```
cimf/
├── config.env              # 实际配置文件（不提交到Git）
├── config.env.sample      # 配置模板（提交到Git）
├── cimf_django/
│   └── settings.py       # Django配置（从环境变量读取）
└── docs/
    └── 技术规范/
        └── A07_环境变量与配置管理规范.md（本文件）
```

## 二、环境变量命名规范

### 2.1 命名规则

| 规则 | 说明 | 示例 |
|------|------|------|
| **前缀** | 使用 `DJANGO_` 作为 Django 配置前缀 | `DJANGO_DEBUG` |
| **大写** | 全部使用大写字母 | `DJANGO_SECRET_KEY` |
| **下划线** | 使用下划线分隔单词 | `DJANGO_ALLOWED_HOSTS` |
| **描述性** | 变量名应清晰描述用途 | `DJANGO_HSTS_SECONDS` |

### 2.2 变量分类

#### 核心配置
| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `DJANGO_SECRET_KEY` | Django 密钥（50+字符） | - | ✅ |
| `DJANGO_DEBUG` | 调试模式（True/False） | `True` | ❌ |
| `DJANGO_ENV` | 环境标识（development/production） | `development` | ❌ |
| `DJANGO_ALLOWED_HOSTS` | 允许访问的主机（逗号分隔） | `localhost,127.0.0.1` | ❌ |

#### 安全配置（生产环境）
| 变量名 | 说明 | 默认值 | 必填（生产） |
|--------|------|--------|--------------|
| `DJANGO_HSTS_SECONDS` | HSTS 过期时间（秒） | `0` | ✅ |
| `DJANGO_HSTS_INCLUDE_SUBDOMAINS` | HSTS 包含子域名 | `false` | ❌ |
| `DJANGO_HSTS_PRELOAD` | HSTS 预加载 | `false` | ❌ |
| `DJANGO_SSL_REDIRECT` | 强制 HTTPS 重定向 | `false` | ✅ |
| `DJANGO_SESSION_COOKIE_SECURE` | Session Cookie HTTPS | `false` | ✅ |
| `DJANGO_CSRF_COOKIE_SECURE` | CSRF Cookie HTTPS | `false` | ✅ |

#### 数据库配置
| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `DB_TYPE` | 数据库类型（sqlite/mysql） | `sqlite` | ❌ |
| `DB_NAME` | 数据库名称 | - | MySQL必填 |
| `DB_USER` | 数据库用户 | - | MySQL必填 |
| `DB_PASSWORD` | 数据库密码 | - | MySQL必填 |
| `DB_HOST` | 数据库主机 | `localhost` | ❌ |
| `DB_PORT` | 数据库端口 | `3306` | ❌ |

#### 访问控制
| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `IP_RESTRICTION_ENABLED` | 是否启用IP限制 | `false` | ❌ |
| `IP_WHITELIST` | IP白名单（逗号分隔） | `127.0.0.1` | IP限制时必填 |

## 三、config.env.sample 规范

### 3.1 文件结构

```bash
# ============================================
# CIMF 管理系统配置文件模板
# ============================================

# ----- 配置分组 -----
VARIABLE_NAME=default_value  # 简短说明
```

### 3.2 注释规范

| 注释类型 | 格式 | 用途 |
|----------|------|------|
| **分组注释** | `# ----- 分组名 -----` | 分隔不同配置区域 |
| **说明注释** | `# 说明文字` | 解释配置用途 |
| **警告注释** | `# 【重要】说明` | 强调安全注意事项 |
| **示例注释** | `# 示例: ...` | 提供配置示例 |
| **行内注释** | `VAR=value  # 说明` | 简短说明 |

### 3.3 示例模板

```bash
# ============================================
# CIMF 管理系统配置文件模板
# ============================================

# ----- Django 核心设置 -----
# 【重要】Django 密钥：生产环境必须使用强随机密钥（50+字符）
# 生成命令: python -c "import secrets; print(secrets.token_urlsafe(50))"
DJANGO_SECRET_KEY=your-secret-key-here-change-in-production

# 调试模式：开发环境=True，生产环境=False
DJANGO_DEBUG=True

# 环境标识：development（开发）或 production（生产）
DJANGO_ENV=development

# 允许访问的主机列表（逗号分隔）
# 生产环境示例: yourdomain.com,www.yourdomain.com
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1

# ----- IP 访问限制 -----
# ...
```

## 四、settings.py 读取规范

### 4.1 读取方法

```python
import os
from dotenv import load_dotenv

# 加载环境变量
load_dotenv(os.path.join(BASE_DIR, 'config.env'))

# 读取配置（带默认值）
DEBUG = os.getenv('DJANGO_DEBUG', 'True').lower() == 'true'

# 读取配置（无默认值，必填）
SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY')
if not SECRET_KEY:
    raise ValueError("必须设置 DJANGO_SECRET_KEY 环境变量！")

# 读取列表类型
ALLOWED_HOSTS_STR = os.getenv('DJANGO_ALLOWED_HOSTS', 'localhost,127.0.0.1')
ALLOWED_HOSTS = [host.strip() for host in ALLOWED_HOSTS_STR.split(',') if host.strip()]
```

### 4.2 类型转换

| 类型 | 方法 | 示例 |
|------|------|------|
| **字符串** | `os.getenv('VAR', 'default')` | `os.getenv('DB_HOST', 'localhost')` |
| **布尔值** | `os.getenv('VAR', 'false').lower() == 'true'` | `DEBUG = ...` |
| **整数** | `int(os.getenv('VAR', '0'))` | `HSTS_SECONDS = ...` |
| **列表** | `os.getenv('VAR', '').split(',')` | `ALLOWED_HOSTS = ...` |

### 4.3 生产环境自动配置

```python
DJANGO_ENV = os.getenv('DJANGO_ENV', 'development')

if DJANGO_ENV == 'production':
    # 自动启用安全配置
    SECURE_SSL_REDIRECT = True
    SESSION_COOKIE_SECURE = True
    CSRF_COOKIE_SECURE = True
    DEBUG = False
    
    # 验证必填配置
    if ALLOWED_HOSTS == ['localhost', '127.0.0.1']:
        raise ValueError("生产环境必须在 DJANGO_ALLOWED_HOSTS 中配置实际域名！")
```

## 五、安全规范

### 5.1 敏感信息管理

| 项目 | 规范 |
|------|------|
| **密钥生成** | 使用 `secrets.token_urlsafe(50)` 生成 |
| **密码存储** | 不存储在代码中，仅通过环境变量 |
| **配置文件** | `config.env` 必须加入 `.gitignore` |
| **日志输出** | 禁止在日志中输出 SECRET_KEY、密码等敏感信息 |

### 5.2 Git 管理

**.gitignore 配置：**
```
# 配置文件
config.env
.env
.env.local

# 保留模板
!config.env.sample
```

## 六、检查清单

### 6.1 开发环境检查

- [ ] `config.env` 文件存在
- [ ] `config.env.sample` 是最新版本
- [ ] `SECRET_KEY` 已设置（开发可用默认值）
- [ ] `DEBUG=True` 用于开发

### 6.2 生产环境检查

- [ ] `DJANGO_ENV=production`
- [ ] `DJANGO_DEBUG=False`
- [ ] `DJANGO_SECRET_KEY` 是强随机密钥（50+字符）
- [ ] `DJANGO_ALLOWED_HOSTS` 包含实际域名
- [ ] `DJANGO_SSL_REDIRECT=True`
- [ ] `DJANGO_SESSION_COOKIE_SECURE=True`
- [ ] `DJANGO_CSRF_COOKIE_SECURE=True`
- [ ] `DJANGO_HSTS_SECONDS >= 31536000`（建议1年）

### 6.3 部署验证

```bash
# 检查配置
./venv/bin/python manage.py check --deploy

# 测试生产配置
DJANGO_ENV=production DJANGO_DEBUG=False ./venv/bin/python manage.py check --deploy
```
