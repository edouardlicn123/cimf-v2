# 模块技术规范

本文档定义 cimf-v2（仙芙CIMF）项目的业务模块系统技术规范，包含模块模型、服务层、视图、模板和权限控制的完整实现指引。

> **重要**：模块系统采用 **模块化设计**，每个业务模块对应一个子应用。模块信息通过 `module.py` 文件定义，并通过 `NodeModule` 注册表统一管理。

> **更新历史**：2026-03-27，目录名从 `nodes` 更新为 `modules`

---

## 一、概述

### 1.1 什么是模块

模块（Module）是 cimf-v2 系统中的业务功能单元，每个模块是一个独立的 Django 子应用，包含完整的模型、服务、视图和模板。

### 1.2 两种初始化方式

| 方式 | 数据来源 | 管理方式 | 状态 |
|------|----------|----------|------|
| 模块化（推荐） | `module.py` + `NodeModule` | 数据库注册表 | 启用 |
| 旧配置（已废弃） | `DEFAULT_NODE_TYPES` | 代码硬编码 | 不推荐 |

**推荐方式**：每个业务模块通过 `module.py` 定义模块信息，然后通过"检查模块"功能扫描并注册到 `NodeModule` 表。

## 三、核心模型

### 3.1 NodeModule - 模块注册表

```python
# core/node/models.py
class NodeModule(models.Model):
    """模块注册表"""
    
    module_id = models.CharField(max_length=50, unique=True, verbose_name='模块ID')
    name = models.CharField(max_length=100, verbose_name='模块名称')
    version = models.CharField(max_length=20, verbose_name='版本号')
    author = models.CharField(max_length=100, blank=True, null=True, verbose_name='作者')
    description = models.TextField(blank=True, null=True, verbose_name='描述')
    path = models.CharField(max_length=200, verbose_name='模块路径')
    
    is_installed = models.BooleanField(default=False, verbose_name='是否已安装')
    is_active = models.BooleanField(default=False, verbose_name='是否启用')
    is_system = models.BooleanField(default=False, verbose_name='是否系统默认模块')
    
    installed_at = models.DateTimeField(null=True, blank=True, verbose_name='安装时间')
    activated_at = models.DateTimeField(null=True, blank=True, verbose_name='启用时间')
    created_at = models.DateTimeField(auto_now_add=True, verbose_name='创建时间')
    updated_at = models.DateTimeField(auto_now=True, verbose_name='更新时间')
    
    class Meta:
        db_table = 'node_modules'
        verbose_name = '模块'
        verbose_name_plural = '模块'
```

## 五、服务层规范

### 5.1 服务命名规范

- `{Module}Service`：模块业务服务，如 `CustomerService`
- 服务类放在 `services.py` 文件中

### 5.2 服务基类方法

```python
# core/node/services.py

class NodeTypeService:
    """节点类型服务"""
    
    @staticmethod
    def get_all() -> List[NodeType]:
        """获取所有启用的节点类型"""
        return list(NodeType.objects.filter(is_active=True))
    
    @staticmethod
    def get_by_slug(slug: str) -> Optional[NodeType]:
        """根据 slug 获取节点类型"""
        return NodeType.objects.filter(slug=slug, is_active=True).first()


class NodeService:
    """节点主表服务"""
    
    @staticmethod
    def create(node_type_slug: str, user, data: Dict[str, Any]) -> Optional[Node]:
        """创建节点"""
        node_type = NodeTypeService.get_by_slug(node_type_slug)
        if not node_type:
            return None
        
        node = Node.objects.create(
            node_type=node_type,
            created_by=user,
            updated_by=user
        )
        return node
```

### 5.3 模块服务示例

```python
# modules/customer/services.py
class CustomerService:
    """客户业务服务"""
    
    @staticmethod
    def get_list(search: Optional[str] = None, **filters) -> List[CustomerFields]:
        """获取客户列表"""
        queryset = CustomerFields.objects.select_related('node')
        
        if search:
            queryset = queryset.filter(customer_name__icontains=search)
        
        return queryset.order_by('-created_at')
    
    @staticmethod
    def create(user, data: Dict[str, Any]) -> CustomerFields:
        """创建客户"""
        node = NodeService.create('customer', user, {})
        
        customer = CustomerFields.objects.create(
            node=node,
            customer_name=data.get('customer_name', ''),
            customer_code=data.get('customer_code'),
            # ... 其他字段
        )
        return customer
```

def check_customer_permission(user, node, permission_type: str):
    """检查客户节点操作权限"""
    if user.is_admin:
        return True, None
    
    is_creator = node.created_by_id == user.id
    
    if is_creator:
        return True, None
    
    perm_map = {
        'view': 'node.customer.view_others',
        'edit': 'node.customer.edit_others',
        'delete': 'node.customer.delete_others',
    }
    
    perm = perm_map.get(permission_type)
    if perm and PermissionService.has_permission(user, perm):
        return True, None
    
    return False, f'您没有权限{permission_type}别人的客户信息'

## 九、INSTALLED_APPS 动态加载

### 9.1 实现原理

在 `settings.py` 中动态扫描 `modules/` 目录：

```python
# cimf_django/settings.py

# 基础应用（必须保留）
_base_apps = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'rest_framework',
    'core',
    'modules',
]

# 动态扫描 modules/ 下的模块
_node_modules = []
_modules_dir = os.path.join(BASE_DIR, 'modules')
if os.path.isdir(_modules_dir):
    for item in os.listdir(_modules_dir):
        module_path = os.path.join(_modules_dir, item)
        if os.path.isdir(module_path) and os.path.exists(os.path.join(module_path, 'module.py')):
            _node_modules.append(f'modules.{item}')

INSTALLED_APPS = _base_apps + _node_modules
```

### 9.2 新增模块流程

```
1. 创建模块目录：modules/{module_id}/
2. 创建 module.py：定义模块信息
3. 创建业务模型：models.py, services.py, views.py 等
4. 执行数据库迁移：makemigrations + migrate
5. 重启服务
```

**无需修改任何配置文件！**

## 十、模块管理

### 10.1 模块服务

`NodeModuleService` 提供模块管理功能：

```python
# core/node/services.py

class NodeModuleService:
    """模块服务"""
    
    MODULES_DIR = 'modules'
    
    @staticmethod
    def scan_modules() -> List[Dict]:
        """扫描所有模块"""
        # 扫描 modules/ 目录下的 module.py
        # 返回模块信息列表
        
    @staticmethod
    def register_module(module_info: Dict) -> NodeModule:
        """注册模块到数据库"""
        
    @staticmethod
    def install_module(module_id: str) -> bool:
        """安装模块（执行迁移）"""
        
    @staticmethod
    def enable_module(module_id: str) -> bool:
        """启用模块"""
        
    @staticmethod
    def disable_module(module_id: str) -> bool:
        """禁用模块"""
```

### 10.2 模块状态

| 状态 | 说明 | 触发操作 |
|------|------|----------|
| **未安装** | 目录存在但未注册 | 扫描模块 |
| **已注册** | 已注册到数据库 | 注册模块 |
| **已安装** | 数据库迁移已完成 | 安装模块 |
| **已启用** | 可正常使用 | 启用模块 |
