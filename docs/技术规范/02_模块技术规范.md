# 模块技术规范

本文档定义 cimf-v2（仙芙CIMF）项目的业务模块系统技术规范。

> **更新历史**：
> - 2026-03-30：添加动态 URL 加载机制，确保模块缺失时系统仍可启动
> - 2026-03-28：新增模块类型分类（node/system）、module_type 字段、扫描逻辑优化
> - 2026-03-27：目录名从 `nodes` 更新为 `modules`

---

## 一、概述

### 1.1 核心概念

| 概念 | 说明 |
|------|------|
| **模块 (Module)** | 独立的业务功能单元，每个模块对应一个 Django 子应用 |
| **节点类型 (NodeType)** | 模块的业务分类，对应事务处理页面的入口 |
| **节点 (Node)** | 节点的公共字段，记录创建人、更新时间等元信息 |
| **字段模型 ({Module}Fields)** | 模块的业务数据模型，存储具体业务字段 |

### 1.2 模块类型

模块通过 `type` 字段分为两种类型：

| 类型 | 说明 | 显示位置 |
|------|------|----------|
| `node` | 事务节点模块，用于业务数据管理 | 事务处理页面（首页、左侧导航） |
| `system` | 系统模块，用于系统功能扩展 | 仪表盘、功能卡片等 |

**设计理念**：
- `node` 类型模块会创建 `NodeType`，出现在事务处理相关页面
- `system` 类型模块不参与节点类型管理，不出现在事务处理页面

## 三、module.py 规范

### 3.1 必选字段

```python
# -*- coding: utf-8 -*-
"""
模块信息文件
定义模块的基本信息，供系统扫描和注册使用。
"""

MODULE_INFO = {
    'id': 'customer',                    # 模块唯一标识
    'name': '客户信息（海外）',            # 显示名称
    'type': 'node',                      # 模块类型 (node=事务节点, system=系统模块)
    'version': '1.0.0',                  # 版本号
}
```

### 3.2 可选字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `author` | string | 作者 |
| `description` | string | 描述 |
| `icon` | string | Bootstrap Icons 类名，如 `bi-people` |
| `dashboard_cards` | list | 仪表盘卡片配置（system 类型模块使用） |
| `taxonomies` | list | 模块词汇表定义，安装时自动创建 |
| `views` | dict | 视图函数映射 |

### 3.3 taxonomies 字段规范

模块可在 `module.py` 中定义词汇表，系统在安装模块时会自动创建这些词汇表及词汇项。

**数据结构**：

```python
'taxonomies': [
    {
        'slug': 'resident_type',           # 必填，词汇表唯一标识（全局唯一）
        'name': '人员类型',                 # 必填，词汇表显示名称
        'items': ['常住人口', '流动人口']    # 必填，词汇项名称列表
    },
]
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `slug` | string | 是 | 词汇表唯一标识，全局唯一 |
| `name` | string | 是 | 词汇表显示名称 |
| `items` | list[string] | 是 | 词汇项名称列表 |

**安装行为**：

| 场景 | 行为 |
|------|------|
| 词汇表不存在 | 创建词汇表 + 创建所有词汇项 |
| 词汇表已存在 | 跳过创建，仅补充缺失的词汇项 |
| 词汇项已存在 | 跳过，不重复创建 |

**示例**：

```python
MODULE_INFO = {
    'id': 'resident_info',
    'name': '居民信息',
    'type': 'node',
    'version': '1.0.0',
    'taxonomies': [
        {'slug': 'resident_type', 'name': '人员类型', 'items': ['常住人口', '流动人口', '暂住人口']},
        {'slug': 'grid', 'name': '所属网格', 'items': ['网格1', '网格2', '网格3']},
    ],
}
```

### 3.5 permissions 字段规范

模块可在 `module.py` 中定义自定义权限，系统会自动将这些权限加入权限管理页面。

**数据结构**：

```python
'permissions': [
    {'key': 'view_others', 'name': '查看别人的内容'},
    {'key': 'edit_others', 'name': '修改别人的内容'},
    {'key': 'delete_others', 'name': '删除别人的内容'},
]
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `key` | string | 是 | 权限标识，最终权限为 `node.{module_id}.{key}` |
| `name` | string | 是 | 权限显示名称 |

**权限生成规则**：

| 权限类型 | 权限标识 | 说明 |
|----------|----------|------|
| 基础权限 | `node.{module_id}.create` | 创建节点（自动添加） |
| 基础权限 | `node.{module_id}.read` | 查看节点（自动添加） |
| 基础权限 | `node.{module_id}.update` | 修改节点（自动添加） |
| 基础权限 | `node.{module_id}.delete` | 删除节点（自动添加） |
| 自定义权限 | `node.{module_id}.{key}` | 从配置读取 |

**示例**：

```python
MODULE_INFO = {
    'id': 'resident_info',
    'name': '居民信息',
    'type': 'node',
    'version': '1.0.0',
    'icon': 'bi-person-vcard',
    'permissions': [
        {'key': 'view_others', 'name': '查看别人的内容'},
        {'key': 'edit_others', 'name': '修改别人的内容'},
        {'key': 'delete_others', 'name': '删除别人的内容'},
    ],
}
```

**安装行为**：

| 场景 | 行为 |
|------|------|
| 模块已启用 | 权限自动出现在权限管理页面 |
| 模块未启用 | 权限不显示 |

## 五、模块管理

### 5.1 模块生命周期

```
未注册 → 注册 → 安装 → 启用
              ↓
            禁用
```

### 5.2 模块状态

| 状态 | 说明 | 触发操作 |
|------|------|----------|
| **未注册** | 目录存在但未注册 | 扫描模块 |
| **已注册** | 已注册到数据库 | 注册模块 |
| **已安装** | 数据库迁移已完成 | 安装模块 |
| **已启用** | 可正常使用 | 启用模块 |
| **已禁用** | 已安装但未启用 | 禁用模块 |

### 5.3 类型过滤逻辑

| 场景 | `type='node'` | `type='system'` |
|------|---------------|-----------------|
| 扫描注册 | ✅ 正常注册 | ✅ 正常注册 |
| 安装模块 | 创建 NodeType | ❌ 不创建 NodeType |
| 启用模块 | 启用节点类型 | ❌ 不操作节点类型 |
| 事务首页 | ✅ 显示 | ❌ 不显示 |

### 5.4 管理功能

| 功能 | 说明 |
|------|------|
| 扫描模块 | 使用 AST 直接解析 module.py，无需重启服务 |
| 安装模块 | 执行迁移，创建 NodeType（仅 node 类型），创建模块词汇表 |
| 启用模块 | 启用节点类型（仅 node 类型） |
| 禁用模块 | 禁用节点类型（仅 node 类型） |

### 6.1 服务层

```python
# modules/customer/services.py
class CustomerService:
    """客户业务服务"""
    
    @staticmethod
    def get_list(search=None, **filters):
        queryset = CustomerFields.objects.select_related('node')
        if search:
            queryset = queryset.filter(customer_name__icontains=search)
        return queryset.order_by('-created_at')
    
    @staticmethod
    def create(user, data):
        node = NodeService.create('customer', user, {})
        return CustomerFields.objects.create(node=node, **data)
```

from django.contrib.auth.decorators import login_required

def node_list(request):

### 6.3.1 动态加载机制

> **重要**：为确保系统在任何模块缺失时都能正常启动，modules/urls.py 必须是**动态安全加载**的。

使用动态加载机制后：
- 模块文件夹被删除时，系统仍能正常启动
- 模块被禁用时，不影响系统运行
- 无需手动更新 URL 路由表（只需更新 `_known_modules` 列表）

**实现代码**：

```python
# modules/urls.py
from django.urls import path, include
from importlib import import_module

def try_include_module(module_slug):
    """尝试动态导入模块 URL，失败则返回空列表"""
    try:
        import_module(f'modules.{module_slug}.urls')
        return [path(f'{module_slug}/', include(f'modules.{module_slug}.urls'))]
    except (ImportError, ModuleNotFoundError, AttributeError):
        return []

# 已知的模块列表（添加新模块时需同步更新）
_known_modules = ['clock', 'customer', 'customer_cn', 'resident_info']
_dynamic_routes = []
for _mod in _known_modules:
    _dynamic_routes.extend(try_include_module(_mod))

# 基础 URL 路由（不变）
_base_urlpatterns = [
    path('', node_views.nodes_index, name='index'),
    # 字段类型、导入导出等固定路由...
]

# 合并所有路由
urlpatterns = _base_urlpatterns + _dynamic_routes
```

**注意事项**：
1. 新增模块时，必须将模块名添加到 `_known_modules` 列表
2. 通用节点路由（`<slug:node_type_slug>/`）必须放在动态路由**之后**，作为 fallback
3. 动态加载失败时使用 `except (ImportError, ModuleNotFoundError, AttributeError)` 捕获所有异常

## 十一、首页卡片系统

### 11.1 概述

首页卡片系统是用户在仪表盘（首页）上看到的快捷入口区域，采用 3×2 网格布局（6个槽位）。

**核心特性**：
- 支持拖拽调整卡片位置
- 位置数据保存在 `SystemSetting` 表（key: `user_dashboard_card_positions`）
- 支持 node 类型和 system 类型模块
- node 类型模块默认显示在卡片区域
- 可在设置页面自定义卡片布局

### 11.2 module.py 配置

在模块的 `module.py` 文件中添加 `dashboard_cards` 配置：

```python
# modules/customer/module.py
# -*- coding: utf-8 -*-

MODULE_INFO = {
    'id': 'customer',
    'name': '客户信息（海外）',
    'type': 'node',
    'version': '1.0.0',
    'author': 'edouardlicn',
    'description': '海外客户信息管理模块',
    'icon': 'bi-people',
    'dashboard_cards': [
        {
            'id': 'customer_card',
            'name': '海外客户',
            'template': 'customer/dashboard_card.html',
        }
    ],
}
```

**配置字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 卡片唯一标识 |
| `name` | string | 是 | 卡片显示名称 |
| `template` | string | 是 | 模板路径（相对于模块 templates 目录） |

### 11.3 卡片模板规范

```

```

| 变量 | 类型 | 说明 |

| `total` | int | 模块数据总数（需实现 `get_count()` 方法） |
| `recent` | int | 最近7天新增数量（需实现 `get_recent_count(7)` 方法） |
| `module_id` | string | 模块ID |
| `position` | int | 卡片位置（1-6） |

### 11.4 统计方法（可选）

如需在卡片上显示统计数据，需在 Service 类中实现以下方法：

```python
# modules/customer/services.py
class CustomerService:
    
    @staticmethod
    def get_count() -> int:
        """获取客户总数"""
        return CustomerFields.objects.count()
    
    @staticmethod
    def get_recent_count(days: int = 7) -> int:
        """获取最近N天新增的客户数量"""
        from django.utils import timezone
        from datetime import timedelta
        start_date = timezone.now() - timedelta(days=days)
        return CustomerFields.objects.filter(created_at__gte=start_date).count()
```

**支持统计的模块**（在 `module_info` 中配置）：
- `customer` → `CustomerService`
- `customer_cn` → `CustomerCnService`

如需支持新的模块统计，需修改 `core/views.py` 中的 `api_dashboard_cards` 函数。
