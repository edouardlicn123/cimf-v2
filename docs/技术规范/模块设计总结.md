# 模块设计总结

## 一、架构概述

本项目采用「主应用 + 子应用」模式组织业务模块，通过模块化设计实现业务功能的解耦和复用。

### 1.1 核心概念

| 概念 | 说明 |
|------|------|
| **模块 (Module)** | 独立的业务功能单元，每个模块对应一个 Django 子应用 |
| **节点类型 (NodeType)** | 模块的业务分类标签，用于数据组织和权限控制 |
| **节点 (Node)** | 节点的公共字段，记录创建人、更新时间等元信息 |
| **字段模型 ({Module}Fields)** | 模块的业务数据模型，存储具体业务字段 |

### 1.2 目录结构

```
cimf-v2/
├── modules/                      # 模块主应用
│   ├── __init__.py
│   ├── apps.py                   # 主应用配置
│   ├── urls.py                   # 路由配置
│   ├── customer/                 # 客户（海外）子模块
│   │   ├── module.py             # 必选：模块信息文件
│   │   ├── models.py             # 字段模型
│   │   ├── services.py           # 业务服务
│   │   ├── views.py              # 视图函数
│   │   ├── urls.py               # 路由
│   │   ├── templates/            # 模板目录
│   │   └── migrations/           # 数据库迁移
│   └── customer_cn/              # 客户（国内）子模块
│       └── ...
├── core/                         # 核心应用
│   └── node/                     # 节点系统
│       ├── models.py             # NodeType, Node, NodeModule
│       ├── services.py           # 公共服务
│       └── views.py              # 通用视图
└── cimf_django/
    └── settings.py               # Django 配置
```

## 二、模块生命周期

### 2.1 模块状态

| 状态 | 说明 | 数据库字段 |
|------|------|------------|
| **未安装** | 目录存在但未注册到数据库 | `is_installed=False` |
| **已安装** | 已注册到 NodeModule 表 | `is_installed=True` |
| **已禁用** | 已安装但未启用 | `is_active=False` |
| **已启用** | 已安装且启用，可正常使用 | `is_active=True` |

### 2.2 状态转换

```
目录创建 → 扫描模块 → 注册模块 → 安装模块 → 启用模块
                              ↓
                        （禁用模块）
```

## 三、核心机制

### 3.1 INSTALLED_APPS 动态加载

在 `settings.py` 中动态扫描 `modules/` 目录：

```python
_node_modules = []
_modules_dir = os.path.join(BASE_DIR, 'modules')
if os.path.isdir(_modules_dir):
    for item in os.listdir(_modules_dir):
        module_path = os.path.join(_modules_dir, item)
        if os.path.isdir(module_path) and os.path.exists(os.path.join(module_path, 'module.py')):
            _node_modules.append(f'modules.{item}')

INSTALLED_APPS = _base_apps + _node_modules
```

**优势**：新增模块无需修改任何配置文件，系统自动发现。

### 3.2 模块信息定义 (module.py)

每个模块根目录必须包含 `module.py` 文件：

```python
MODULE_INFO = {
    'id': 'customer',              # 模块唯一标识
    'name': '客户信息（海外）',     # 显示名称
    'version': '1.0.0',            # 版本号
    'author': 'edouardlicn',       # 作者
    'description': '...',           # 描述
    'models': ['CustomerFields'],  # 数据模型
    'dependencies': [],            # 依赖模块
    'icon': 'bi-people',           # 图标
}
```

### 3.3 NodeModule 注册表

数据库中的模块注册表，记录所有已发现的模块：

| 字段 | 说明 |
|------|------|
| `module_id` | 模块唯一标识 |
| `name` | 模块名称 |
| `version` | 版本号 |
| `path` | 模块目录名 |
| `is_installed` | 是否已安装 |
| `is_active` | 是否启用 |
| `is_system` | 是否系统默认模块 |

### 3.4 模块与 NodeType 同步

模块启用时自动同步创建对应的 NodeType：

- `module_id` → `NodeType.slug`
- `name` → `NodeType.name`
- `description` → `NodeType.description`
- `is_active` → `NodeType.is_active`

## 五、数据模型

### 5.1 Node 公共字段

所有节点共用的字段：

```python
class Node(models.Model):
    node_type = ForeignKey(NodeType)    # 节点类型
    created_by = ForeignKey(User)       # 创建人
    updated_by = ForeignKey(User)       # 更新人
    created_at = DateTimeField          # 创建时间
    updated_at = DateTimeField          # 更新时间
```

### 5.2 模块字段模型

每个模块自己的字段表：

```python
class CustomerFields(models.Model):
    node = OneToOneField(Node)          # 关联 Node
    customer_name = CharField            # 客户名称
    customer_code = CharField            # 客户代码
    # ... 其他业务字段
```

## 六、路由设计

### 6.1 URL 命名空间

- **模块路由**：`modules:{子模块}:{action}`
- **节点路由**：`modules:node_{action}`

### 6.2 典型路由

```
/modules/                    # 模块总览
/modules/types/              # 节点类型列表
/modules/modules/            # 模块管理
/modules/{slug}/             # 节点列表
/modules/{slug}/create/     # 创建节点
/modules/{slug}/{id}/        # 查看节点
/modules/{slug}/{id}/edit/   # 编辑节点
/modules/{slug}/{id}/delete/ # 删除节点
```

## 七、权限控制

### 7.1 模块级别权限

| 权限 | 说明 |
|------|------|
| `module.view` | 查看模块列表 |
| `module.install` | 安装模块 |
| `module.enable` | 启用模块 |
| `module.disable` | 禁用模块 |

### 7.2 节点级别权限

| 权限 | 说明 |
|------|------|
| `node.{slug}.view` | 查看节点 |
| `node.{slug}.create` | 创建节点 |
| `node.{slug}.edit` | 编辑节点 |
| `node.{slug}.delete` | 删除节点 |
| `node.{slug}.view_others` | 查看他人节点 |
| `node.{slug}.edit_others` | 编辑他人节点 |
| `node.{slug}.delete_others` | 删除他人节点 |
