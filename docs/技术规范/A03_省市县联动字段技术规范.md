## 1. 概述

### 1.1 系统目标

为中国行政区划（省-市-区）提供三级联动选择能力，支持在表单中快速选择地区，数据存储为 JSON 格式。

### 1.2 数据来源

- **主数据源**：https://github.com/modood/Administrative-divisions-of-China
- **数据格式**：pca-code.json（省市区三级联动数据）
- **数据量**：31 个省份、342 个城市、3056 个区县
- **本地文件**：`core/data/china_regions.json`

### 1.3 技术选型

- **后端框架**：Django 6.0
- **前端框架**：原生 JavaScript + Bootstrap 5
- **数据存储**：独立数据库表（非 Taxonomy 系统）
- **API 风格**：RESTful JSON API

## 3. 数据模型

### 3.1 ChinaRegion 模型

**文件**：`core/models.py`

```python
class ChinaRegion(models.Model):
    """中国行政区划"""
    
    LEVEL_CHOICES = [
        (1, '省级'),
        (2, '地级市'),
        (3, '县/区'),
    ]
    
    code = models.CharField(max_length=6, unique=True, verbose_name='行政区划代码')
    name = models.CharField(max_length=100, verbose_name='名称')
    level = models.IntegerField(choices=LEVEL_CHOICES, verbose_name='层级')
    parent = models.ForeignKey(
        'self',
        on_delete=models.CASCADE,
        null=True, blank=True,
        related_name='children',
        verbose_name='父级行政区划'
    )
    created_at = models.DateTimeField(auto_now_add=True, verbose_name='创建时间')
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| code | CharField(6) | 行政区划代码，如 440305 |
| name | CharField(100) | 地区名称，如 "南山区" |
| level | IntegerField | 1=省/直辖市, 2=市, 3=区/县 |
| parent | ForeignKey | 自关联，指向父级地区 |
| created_at | DateTimeField | 创建时间 |

### 3.2 层级关系

```
广东省 (code=44, level=1)
  ├── 广州市 (code=4401, level=2, parent=44)
  │     ├── 天河区 (code=440106, level=3, parent=4401)
  │     ├── 番禺区 (code=440113, level=3, parent=4401)
  │     └── ...
  ├── 深圳市 (code=4403, level=2, parent=44)
  │     ├── 南山区 (code=440305, level=3, parent=4403)
  │     └── ...
  └── ...
```

## 4. 服务层

### 4.1 ChinaRegionService

**文件**：`core/services/china_region_service.py`

**主要方法**：

| 方法 | 说明 | 返回值 |
|------|------|--------|
| `import_from_file(file_path)` | 从本地 JSON 文件导入数据 | `{'success': True, 'count': 3429, ...}` |
| `import_from_url(url)` | 从网络导入数据（自动重试备用源） | `{'success': True, 'count': 3429, ...}` |
| `download_to_file(url)` | 从网络下载数据保存到本地文件 | `{'success': True, 'file_path': '...', ...}` |
| `get_provinces()` | 获取所有省份 | `List[ChinaRegion]` |
| `get_cities(province_code)` | 获取省份下的所有城市 | `List[ChinaRegion]` |
| `get_districts(city_code)` | 获取城市下的所有区县 | `List[ChinaRegion]` |
| `get_tree()` | 获取完整树形结构 | `List[Dict]` |
| `search(keyword)` | 搜索地区 | `List[ChinaRegion]` |
| `get_full_path(code)` | 获取完整路径 | `"广东省 - 深圳市 - 南山区"` |
| `get_stats()` | 获取统计信息 | `{'provinces': 31, 'cities': 342, ...}` |

### 4.2 数据导入优化

使用 `bulk_create` 批量导入数据，提升性能：

```python
# 分三批导入：省 → 市 → 区
with transaction.atomic():
    ChinaRegion.objects.all().delete()
    
    # 第一批：省份
    provinces = [ChinaRegion(code=p['code'], name=p['name'], level=1) ...]
    ChinaRegion.objects.bulk_create(provinces)
    
    # 第二批：城市
    cities = [ChinaRegion(code=c['code'], name=c['name'], level=2, parent=province) ...]
    ChinaRegion.objects.bulk_create(cities)
    
    # 第三批：区县
    districts = [ChinaRegion(code=d['code'], name=d['name'], level=3, parent=city) ...]
    ChinaRegion.objects.bulk_create(districts)
```

**导入结果**：
- 省份：31 条
- 城市：342 条
- 区县：3056 条
- 总计：3429 条

## 5. API 接口

### 5.1 接口列表

| 接口 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/api/v1/regions/provinces/` | GET | - | 获取所有省份 |
| `/api/v1/regions/cities/` | GET | `province` | 获取指定省份的城市 |
| `/api/v1/regions/districts/` | GET | `city` | 获取指定城市的区县 |
| `/api/v1/regions/search/` | GET | `q` | 搜索地区 |
| `/api/v1/regions/path/` | GET | `code` | 获取完整路径 |
| `/api/v1/regions/stats/` | GET | - | 获取统计信息 |
| `/api/v1/regions/import/` | POST | - | 导入数据（需管理员） |

### 5.2 响应格式

所有 API 统一响应格式：

```json
{
  "success": true,
  "data": [...]
}
```

**示例：获取省份**

```bash
GET /api/v1/regions/provinces/

Response:
{
  "success": true,
  "data": [
    {"code": "11", "name": "北京市"},
    {"code": "12", "name": "天津市"},
    {"code": "13", "name": "河北省"},
    ...
  ]
}
```

**示例：获取完整路径**

```bash
GET /api/regions/path/?code=440305

Response:
{
  "success": true,
  "data": {
    "code": "440305",
    "path": "广东省 - 深圳市 - 南山区"
  }
}
```

```html
<div class="region-select-widget">
    <div class="row">
        <div class="col-md-4">

                <option value="">请选择省份</option>
            </select>
        </div>
        <div class="col-md-4">
            <select class="form-select region-city" disabled>
                <option value="">请先选择省份</option>
            </select>
        </div>
        <div class="col-md-4">
            <select class="form-select region-district" disabled>
                <option value="">请先选择城市</option>
            </select>

    </div>
    <input type="hidden" name="{name}" value='{"province": "", "city": "", "district": ""}'>
</div>
```

### 6.4 前端 JavaScript

**文件**：`static/js/region_select.js`

**工作流程**：

1. 页面加载时自动初始化所有 `.region-select-widget`
2. 调用省份 API 加载省份列表
3. 用户选择省份后，启用城市下拉框，调用城市 API
4. 用户选择城市后，启用区县下拉框，调用区县 API
5. 用户选择区县后，更新隐藏 input 的值


前端 JavaScript 需要处理统一的返回格式 `{success: true, data: [...]}`：

```javascript
fetch('/api/v1/regions/provinces/')
    .then(r => r.json())
    .then(json => {
        if (json.success && json.data) {
            // 处理数据
            json.data.forEach(p => {...});
        }
```

- 所有 API 接口无需登录即可访问（数据公开）

## 12. 数据维护流程

### 12.1 数据架构

```
┌─────────────────────────────────────────────────────────────────┐
│                       数据维护架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐  │
│  │  GitHub 源   │ ───▶ │ 本地 JSON    │ ───▶ │   数据库     │  │
│  │  (网络下载)  │      │ china_regions│      │ ChinaRegion │  │
│  └──────────────┘      └──────────────┘      └──────────────┘  │
│         │                    ▲                    ▲            │
│         │                    │                    │            │
│  run.sh 维护菜单      init_db.py 初始化      API 查询         │
│  选项 5: 下载更新     --with-data             前端使用         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 12.2 初始化流程

系统初始化时，数据从本地文件导入：

```bash
# 使用 init_db.py 初始化
./venv/bin/python init_db.py --with-data

# 内部调用
ChinaRegionService.import_from_file()
```

**流程说明：**
1. 系统初始化阶段调用 `ChinaRegionService.import_from_file()`
2. 从 `core/data/china_regions.json` 读取数据
3. 使用 `bulk_create` 批量写入数据库
4. 数据量：3429 条（31 省 + 342 市 + 3056 区）

### 12.3 维护更新流程

当需要更新省市区数据时，使用 run.sh 维护菜单：

```bash
# 进入维护菜单
./run.sh 3

# 选择选项 5：下载/更新省市区数据（从网络更新本地文件）
```

**流程说明：**
1. 从网络（GitHub + 备用镜像）下载最新数据
2. 保存到 `core/data/china_regions.json`
3. 自动更新数据库中的数据

### 12.4 run.sh 维护菜单

```
维护菜单：
  1 → 数据库备份
  2 → 清理缓存
  3 → 查看环境变量
  4 → 杀死服务器进程
  5 → 下载/更新省市区数据（从网络更新本地文件）  ← 新增
  0 → 返回主菜单
```

## 13. 故障排除

### 13.1 网络超时

**问题**：从网络下载数据时超时

**解决方案**：
1. 系统会自动尝试备用镜像源（ghproxy、gitee）
2. 如全部失败，可手动下载并替换 `core/data/china_regions.json`
3. 然后使用 `init_db.py` 重新导入

### 13.2 数据不一致

**问题**：本地文件与数据库数据不同步

**解决方案**：
```bash
# 重新导入本地文件到数据库
./venv/bin/python -c "
from core.services.china_region_service import ChinaRegionService
result = ChinaRegionService.import_from_file()
print(result)
"
```

### 13.3 初始化后无数据

**问题**：系统初始化后没有省市区数据

**解决方案**：
```bash
# 使用 run.sh 维护菜单重新下载并导入
./run.sh 3 → 5
```
