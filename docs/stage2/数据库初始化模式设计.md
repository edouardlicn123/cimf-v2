# 数据库初始化模式设计方案

## 目标

设计一个灵活的数据库初始化架构，同时支持：
1. **开发环境**：自动创建表 + 种子数据
2. **生产环境**：使用 Flask-Migrate 迁移管理

## 背景

当前项目使用 `init_schema.py` 脚本手动初始化数据库，不够方便。希望在应用启动时根据配置自动处理。

## 方案设计

### 1. 配置项设计

在 `config.py` 的 `BaseConfig` 中添加以下配置：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| DB_INIT_MODE | string | 'none' | 初始化模式：none/auto/schema_only |
| DB_SEED_DATA | bool | true | 是否初始化种子数据（仅 auto 模式有效） |

**模式说明：**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| none | 不自动初始化 | 生产环境，使用 Flask-Migrate |
| auto | 自动创建表 + 种子数据 | 开发环境，快速启动 |
| schema_only | 仅创建表结构 | 已有种子数据，仅需建表 |

### 2. 各环境的默认配置

| 环境 | DB_INIT_MODE | DB_SEED_DATA |
|------|--------------|--------------|
| development | auto | true |
| testing | auto | false |
| production | none | false |

### 3. 代码实现

#### 3.1 config.py 新增配置

```python
# =============================================
# 数据库初始化模式控制
# =============================================
# DB_INIT_MODE: 控制应用启动时的数据库初始化行为
#   - 'none': 不自动初始化（使用 Flask-Migrate）
#   - 'auto': 自动创建表 + 种子数据
#   - 'schema_only': 仅创建表结构
#
# 环境变量: DB_INIT_MODE
# 默认值: none（生产安全优先）
DB_INIT_MODE = os.environ.get('DB_INIT_MODE', 'none')

# DB_SEED_DATA: 是否初始化种子数据（仅 DB_INIT_MODE='auto' 时有效）
#   - true: 初始化种子数据（管理员账号、系统设置、词汇表等）
#   - false: 仅创建表结构
#
# 环境变量: DB_SEED_DATA
# 默认值: true
DB_SEED_DATA = os.environ.get('DB_SEED_DATA', 'true').lower() in ('true', '1', 'yes')
```

#### 3.2 app/__init__.py 新增初始化函数

在 `create_app()` 中添加 `init_database()` 调用：

```python
def init_database(app):
    """根据配置模式初始化数据库
    
    初始化顺序：
    1. 创建数据库表（schema_only/auto 模式）
    2. 初始化种子数据（仅 auto 模式 + DB_SEED_DATA=true）
       - 系统设置（SettingsService）
       - 词汇表数据（TaxonomyService）
    """
    mode = app.config.get('DB_INIT_MODE', 'none')
    
    if mode == 'none':
        app.logger.info("数据库初始化已禁用（DB_INIT_MODE=none），请使用 Flask-Migrate 管理")
        return
    
    with app.app_context():
        # 步骤 1: 创建表
        if mode in ('auto', 'schema_only'):
            db.create_all()
            app.logger.info("数据库表创建完成")
        
        # 步骤 2: 种子数据
        if mode == 'auto' and app.config.get('DB_SEED_DATA'):
            # 初始化系统设置
            from app.services.settings_service import SettingsService
            SettingsService.reset_to_default()
            app.logger.info("系统设置初始化完成")
            
            # 初始化词汇表
            from app.modules.taxonomy.service import TaxonomyService
            TaxonomyService.init_default_taxonomies()
            app.logger.info("词汇表数据初始化完成")
```

#### 3.3 各环境默认配置

```python
class DevelopmentConfig(BaseConfig):
    ENV = 'development'
    DEBUG = True
    DB_INIT_MODE = 'auto'      # 开发环境自动初始化
    DB_SEED_DATA = True        # 开发环境包含种子数据


class ProductionConfig(BaseConfig):
    ENV = 'production'
    DEBUG = False
    DB_INIT_MODE = 'none'      # 生产环境禁用自动初始化
    DB_SEED_DATA = False
```

## 使用方式

### 开发环境

```bash
# 方式 1: 默认配置（自动初始化）
python run.py

# 方式 2: 覆盖环境变量
DB_INIT_MODE=auto DB_SEED_DATA=true python run.py

# 方式 3: 仅建表，不含种子数据
DB_INIT_MODE=schema_only python run.py
```

### 生产环境

```bash
# 方式 1: 禁用自动初始化，使用 Flask-Migrate
DB_INIT_MODE=none python run.py

# 首次部署时手动运行迁移
flask db init
flask db migrate -m "initial migration"
flask db upgrade

# 或使用 init_schema.py（如果已有数据）
python init_schema.py --with-data
```

## 注意事项

1. 生产环境默认禁用自动初始化，确保数据安全
2. 种子数据初始化前应检查是否已存在，避免重复插入
3. 未来可扩展更多种子数据类型（如权限、角色等）
