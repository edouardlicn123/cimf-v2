# 客户信息（国内）模块技术规范

本文档定义客户信息（国内）模块的技术规范，基于 Node 模块规范实现。

---

## 一、概述

### 1.1 模块背景

客户信息（国内）是针对中国国内客户的客户管理模块，与现有的客户信息（海外）模块独立运行。

### 1.2 模块差异

| 特性 | 客户信息（海外） | 客户信息（国内） |
|------|-----------------|-----------------|
| Slug | customer | customer_cn |
| 国家字段 | 有（country） | 无 |
| 省份字段 | province（文本） | region（省市县联动） |
| 微信号 | 无 | 有 |
| 钉钉号 | 无 | 有 |

```python
class CustomerCnFields(models.Model):
```

### 4.2 权限检查

参考客户信息（海外）的权限检查函数，创建 `check_customer_cn_permission`：

```python
def check_customer_cn_permission(user, node, permission_type: str) -> tuple:
    """
    检查国内客户节点操作权限
    """
    if user.is_admin:
        return True, None
    
    is_creator = node.created_by_id == user.id
    
    if is_creator:
        return True, None
    
    perm_map = {
        'view': 'node.customer_cn.view_others',
        'edit': 'node.customer_cn.edit_others',
        'delete': 'node.customer_cn.delete_others',
    }
    
    perm = perm_map.get(permission_type)
    if perm and PermissionService.has_permission(user, perm):
        return True, None
    
    return False, f'您没有权限{permission_type}别人的国内客户信息'
```

```python
def get_region_display(region: dict) -> str:
    """获取省市区显示文本"""
    if not region:
        return ''
    
    parts = []
    if region.get('province_name'):
        parts.append(region['province_name'])
    if region.get('city_name'):
        parts.append(region['city_name'])
    if region.get('district_name'):
        parts.append(region['district_name'])
    
    return ' '.join(parts)
```

## 九、与客户信息（海外）的关系

### 9.1 数据独立

- 两套数据存储在不同的数据表：
  - 海外：`customer_fields`
  - 国内：`customer_cn_fields`
- 两套 Node 记录，node_type 不同

### 9.2 权限独立

- 权限代码不同：`node.customer.*` vs `node.customer_cn.*`
- 管理员可以分别控制不同模块的权限

### 9.3 菜单显示

左侧导航栏会同时显示两个菜单项：

```
事务总览
├── 客户信息（海外）
└── 客户信息（国内）
```
