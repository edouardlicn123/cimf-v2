# 权限系统设计

## 概述

本文档描述 CIMF 系统的权限设计，包括两个独立概念：**is_admin**（系统管理员）和 **role**（业务角色）。

---

## 权限概念

### 1. is_admin - 系统管理员

- **作用**：控制用户是否能访问**系统管理页面**
- **判断条件**：`user.is_admin == True`
- **可访问的页面**：
  - 系统设置
  - 人员管理
  - 权限管理
  - Cron 调度管理

### 2. role - 业务角色

- **作用**：控制**非admin用户**的业务权限
- **三个值**：
  | 角色 | 数据库值 | 显示名称 |
  |------|----------|----------|
  | 一类用户 | manager | 一类用户 |
  | 二类用户 | leader | 二类用户 |
  | 三类用户 | employee | 三类用户 |

---

## 权限逻辑

| 用户类型 | is_admin | role | 系统管理页面访问 | 业务权限 |
|---------|----------|------|----------------|---------|
| 超级管理员 | True | 任意 | 可访问 | 拥有所有权限 `['*']` |
| 普通用户 | False | manager | 不可访问 | 从权限配置读取 |
| 普通用户 | False | leader | 不可访问 | 从权限配置读取 |
| 普通用户 | False | employee | 不可访问 | 从权限配置读取 |

**说明**：
- admin 用户（is_admin=True）天然拥有所有权限，与角色配置无关
- 非 admin 用户只能通过权限管理页面配置其业务权限

---

## 权限检查方法

### 核心方法

**文件**：`core/services/permission_service.py`

```python
@staticmethod
def can_access_admin(user: User) -> bool:
    """检查用户是否可以访问后台"""
    return user.is_admin == True
```

### 视图中的使用

在需要 admin 权限的视图函数中，使用 `can_access_admin` 进行检查：

```python
@login_required
def system_settings(request):
    """系统设置页面"""
    if not PermissionService.can_access_admin(request.user):
        messages.error(request, '需要系统管理员权限访问该页面')
        return redirect('core:dashboard')
    # ... 视图逻辑
```

## 需要 admin 权限的页面

以下页面需要 `is_admin=True` 才能访问：

| 视图函数 | URL | 页面名称 |
|----------|-----|----------|
| `system_settings` | `/system/settings/` | 系统设置 |

## 代码规范

### 视图函数权限标注

所有需要 admin 权限的视图函数，必须在顶部添加权限检查：

```python
@login_required
def system_settings(request):
    """系统设置页面"""
    # 必须添加权限检查
    if not PermissionService.can_access_admin(request.user):
        messages.error(request, '需要系统管理员权限访问该页面')
        return redirect('core:dashboard')
    
    # 视图逻辑...
```

### 权限检查模式

```python
if not PermissionService.can_access_admin(request.user):
    messages.error(request, '需要系统管理员权限访问该页面')
    return redirect('core:dashboard')
```

## 权限管理页面说明

权限管理页面（`system_permissions`）管理的是**非admin用户的角色权限**，而非系统管理权限。

权限管理页面包含：
1. **角色名称设置** - 修改一类用户、二类用户、三类用户的显示名称
2. **节点权限卡片** - 配置各角色对节点的业务权限

**不包括**系统管理页面的访问权限（由 is_admin 控制）。
