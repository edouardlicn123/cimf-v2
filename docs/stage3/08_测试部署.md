### 8.4 部署配置

#### 8.4.1 生产环境设置

```python
# cimf_django/settings.py

DEBUG = False
ALLOWED_HOSTS = ['your-domain.com']

# 安全设置
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
```

#### 8.4.2 WSGI/ASGI 配置

```python
# cimf_django/wsgi.py
import os
from django.core.wsgi import get_wsgi_application

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'cimf_django.settings')
application = get_wsgi_application()
```

#### 8.4.3 Gunicorn 配置

```bash
# 安装 gunicorn
pip install gunicorn

# 运行
gunicorn cimf_django.wsgi:application --bind 0.0.0.0:8000
```

#### 8.4.4 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location /static/ {
        alias /path/to/staticfiles/;
    }
    
    location /media/ {
        alias /path/to/media/;
    }
    
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 8.5 数据迁移（如需要）

#### 8.5.1 从 Flask 迁移数据

```bash
# 1. 导出 Flask 数据
python manage.py dumpdata --format=yaml > data.yaml

# 2. 运行 Django migrations
python manage.py migrate

# 3. 导入数据
python manage.py loaddata data.yaml
```

#### 8.5.2 直接使用现有数据库

由于使用相同的 SQLite 数据库，可直接配置：

```python

    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': '/home/edo/cimf-flask/instance/site.db',
    }
}
```
