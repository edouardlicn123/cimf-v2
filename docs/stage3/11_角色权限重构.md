# 角色权限重构计划

## 需求概述

1. 将 admin 角色改为一类用户（manager），但系统设置默认叫法叫"经理"
2. 在权限管理页增加一类用户的名称修改功能
3. 在权限管理页增加一类用户的权限修改功能（客户信息CRUD等业务权限）
4. is_admin 作为独立权限，控制系统关键页面访问（系统设置、人员管理、权限管理、Cron调度）
5. 顶部菜单"系统管理"链接仅 is_admin=True 的用户可见

---

## 设计说明

### 角色（role）- 纯业务角色

| 角色 | 数据库值 | 显示名称 |
|------|----------|----------|
| 一类用户 | manager | 一类用户 |
| 二类用户 | leader | 二类用户 |
| 三类用户 | employee | 三类用户 |

### is_admin - 系统管理权限

| is_admin | 说明 |
|----------|------|
| True | 可访问系统设置、人员管理、权限管理、Cron调度 |
| False | 不可访问 |

### 权限管理页面

- 角色名称设置：一类用户、二类用户、三类用户
- 角色权限设置：一类用户、二类用户、三类用户的业务权限
- 系统关键页面访问权限由 is_admin 控制（不在权限管理页设置）
