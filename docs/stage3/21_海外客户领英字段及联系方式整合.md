## 一、功能概述

为海外客户模块增加领英（LinkedIn）字段，并将联系方式信息整合到一个统一的卡片中显示。

### 1.1 需求背景

1. **领英字段需求**：用户需要保存客户的 LinkedIn 主页链接，并在详情页提供可点击链接
2. **联系方式整合**：当前联系方式分散在两个卡片（联系方式1、联系方式2）中，不便于用户查看和编辑

### 1.2 预期效果

**编辑页**：
- 合并"联系方式1"和"联系方式2"为单一的"联系方式"卡片
- 新增"领英"输入框

**详情页**：
- 合并"联系方式1"和"联系方式2"为单一的"联系方式"卡片
- 领英字段显示为可点击链接（新窗口打开）

### 1.3 涉及字段类型

根据 `docs/技术规范/02_Node模块技术规范.md` 中的系统预定义字段类型：

| 字段 | 对应系统字段类型 | 说明 |
|------|------------------|------|
| linkedin | `link` | 链接类型，存储 URL |

### 4.2 update 方法

`update` 方法使用动态赋值，无需修改：

```python
@staticmethod
def update(customer_id: int, user, data: Dict[str, Any]) -> Optional[CustomerFields]:
    """更新客户"""
    customer = CustomerFields.objects.filter(id=customer_id).first()
    if not customer:
        return None
    
    NodeService.update(customer.node_id, user, {})
    
    for key, value in data.items():
        if hasattr(customer, key) and key != 'id' and key != 'node':
            setattr(customer, key, value)
    
    customer.save()
    return customer
```

### 4.3 get_exportable_fields 方法

### 4.4 get_list 方法（如需搜索领英）

如需支持按领英搜索，可扩展 `get_list` 方法：

```python
@staticmethod
def get_list(search: Optional[str] = None, customer_type_id: Optional[int] = None,
             customer_level_id: Optional[int] = None, user=None) -> List[CustomerFields]:
    """获取客户列表"""
    queryset = CustomerFields.objects.select_related(
        'customer_type', 'customer_level', 'country', 'enterprise_type', 'node__created_by'
    )
    
    # ... 其他筛选条件
    
    if search:
        queryset = queryset.filter(
            Q(customer_name__icontains=search) |
            Q(enterprise_name__icontains=search) |
            Q(phone1__icontains=search) |
            Q(phone2__icontains=search) |
            Q(linkedin__icontains=search)  # ← 新增
        )
    
    return queryset.order_by('-created_at')
```
